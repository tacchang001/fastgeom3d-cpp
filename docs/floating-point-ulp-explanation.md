# 浮動小数点精度と ULP の説明

## 背景

[tests/test_intersections.cpp](tests/test_intersections.cpp) には、`double` と `long double` の違いによって交差判定の答えが変わりうることを示すテスト `CircleIntersectionCanChangeWithFloatingPointPrecision` があります。

この文書では、なぜその現象が `1e16` 付近で起きるのかを、ULP の観点から説明します。

## ULP とは何か

ULP は Units in the Last Place の略で、その値の近くで浮動小数点数がどれだけ細かく数を表せるかを示す刻み幅です。

`double` は約 53 ビットの精度を持つため、値が $2^e$ 程度の大きさにあるとき、ULP は概ね次のようになります。

$$
\mathrm{ULP} \approx 2^{e-52}
$$

`1e16` はおおよそ $2^{53}$ に近い値なので、この付近では

$$
\mathrm{ULP}(1e16) \approx 2^{53-52} = 2
$$

となります。つまり、`1e16` 付近の `double` は 2 刻み程度でしか値を区別できません。

## テストで使っている値

テストで意図している配置は次の通りです。

- 円 A の中心: `x = 1e16`
- 円 B の中心: `x = 1e16 + 2.25`
- 半径: どちらも `1`

数学的には中心間距離は `2.25` です。半径和は `2` なので、厳密には非交差です。

```text
long double の世界

1e16                     1e16+2                  1e16+2.25
 |--------------------------|------|
 A中心                      接する位置             B中心(本来)

中心距離 = 2.25 > 2
=> 非交差
```

## double に丸めると何が起きるか

`double` では `1e16` 付近の ULP が約 `2` なので、`1e16 + 2.25` をそのまま表現できません。この値は最も近い表現可能値に丸められます。

この近傍では、ざっくり次のような値しかありません。

```text
double の世界（ULP ≈ 2）

...      1e16        1e16+2        1e16+4      ...
         |------------|-------------|

本来の B中心 = 1e16+2.25
                 ↓ 丸め
              1e16+2
```

その結果、ライブラリが `double` として受け取る中心座標は次のようになります。

- 円 A の中心: `1e16`
- 円 B の中心: `1e16 + 2`

よって、中心間距離は `2` と見なされます。

```text
double でライブラリが見る配置

1e16                     1e16+2
 |--------------------------|
 A中心                      B中心(丸め後)

中心距離 = 2
=> 接触
```

## なぜ答えが変わるのか

この違いは、交差判定アルゴリズムそのものの違いではありません。もっと手前、つまり入力値を `double` に変換した時点で情報が失われることが原因です。

- `long double` で見れば中心距離は `2.25` なので非交差
- `double` に丸めると中心距離が `2.0` になる
- ライブラリは `double` ベースなので、結果は `TOUCH`

つまり、判定式の中で差が出ているのではなく、座標を保持する精度の違いによって、判定対象そのものが変わっています。

## 直感的な見方

`1e16` 付近の `double` は、細かい目盛りの定規ではなく、最小目盛りが `2` の粗い定規だと思うと分かりやすいです。

- `2.25` の差を見たい
- しかし定規の最小目盛りは `2`
- そのため `2.25` は `2` に丸められる
- 結果として「本当は離れているのに、ちょうど接している」と判定される

## まとめ

- `double` は値が大きくなるほど刻み幅が粗くなる
- `1e16` 付近では ULP が約 `2`
- そのため `1e16 + 2.25` のような差は保持できない
- このライブラリは `double` ベースなので、巨大座標では `long double` の直感と答えが変わることがある
